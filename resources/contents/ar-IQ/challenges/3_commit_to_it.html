انشئ ملف في مستودع البيانات الجديد, غير محتوى الملف, و ثبت التغيير بواسطة Git

تثبيت التعديلات

التعديلات هي لب ال Git. هي تمثل اللحظات التي تخزن وتوصف العمل الذي قمت به. هي العلامات في الجدول الزمني لتاريخ مشروعك.

أنشئ ملفا جديدا

لديك الأن مستودع بيانات جاهز, دعنا نظيف بعض الملفات لها

افتح محرر النصوص وانشئ ملفا خاليا. اكتب بعض الكتابات, مثلا "مرحبا!", واخزن الملف باسم “readme.txt” في المجلد “hello-world” الذي انشأته في المهمة السابقة.

تحقق من الحالة + أضف التعديل وثبته

تحقق من ال الحالة لمستودع البيانات لتكتشف اذا كان هناك تعديلات. انت تعرف إنك قد عدلت شيئا ما ولكن هل Git تعرف أيضا؟

تأكد انك ما زلت ضمن مجلد “hello-world” عندما تعطي هذه الايعازات. استخدم Git لترى من تغير في مستودعة بياناتك.

أولا, تحقق من الحالة:

git status

Git ستخبرك ان ملفا قد اضيف

ثم أضف الملف التي أنشأته لكي يصبح جزءا من التعديلات التي سوف تثبتها باستخدام Git

git add readme.txt

وأخيرا, ثبت هذه التعديلات في تاريخ مستودع البيانات مع رسالة قصيرة تصف التغيير

git commit -m "Created readme"

خطوة: اصنع تغييرات اخرى

الان أضف سطرا اخر الى “readme.txt” واخزن الملف مرة اخرى

في ال terminal, يمكنك تفحص الفروقات في الملف بين حالته الحالية والسابقة

اخبر Git بان تظهر لك الفروقات:

git diff

الان مع ما تعلمته, ثبت التعديل الاخير
